# firebase_auth_kit — Example App

This is the example app for the [`firebase_auth_kit`](https://pub.dev/packages/firebase_auth_kit) package.  
It shows how to integrate the pre-built login and signup screens into a real Flutter app using a `Stack`-based layout so the background is fully customisable.

---

## What this example demonstrates

- Login screen with **Email/Password**, **Google Sign-In**, and **Phone OTP**
- Signup screen with the same three methods
- **Stack layout** — a gradient background sits behind the auth screens
- Transparent header — no separate blue section, everything blends into one screen
- Navigation between login → signup → home without named routes
- How to use optional fields like `headerLogoWidget`, `colorScheme`, `headerTitleStyle`

## Firebase Setup (required before running)

### Step 1 — Create a Firebase project

1. Go to [console.firebase.google.com](https://console.firebase.google.com)
2. Click **Add project** and follow the steps

### Step 2 — Enable Authentication methods

In your Firebase project:

1. Go to **Authentication → Sign-in method**
2. Enable:
   - **Email/Password**
   - **Google**
   - **Phone**

### Step 3 — Enable Firestore

1. Go to **Firestore Database**
2. Click **Create database**
3. Start in **test mode** (for development)

### Step 4 — Connect Flutter to Firebase

Install the FlutterFire CLI:

```sh
dart pub global activate flutterfire_cli
```

Run it inside the `example/` folder:

```sh
cd example
flutterfire configure
```

This generates `lib/firebase_options.dart` automatically.

### Step 5 — Run the app

```sh
flutter run
```

## How the Stack layout works

The screen is built as a `Stack` with two layers:

```
Scaffold
└── Stack (StackFit.expand)
    ├── Layer 0 — Background (gradient / image / color / any widget)
    └── Layer 1 — AuthLoginScreen (transparent, sits on top)
```

**Layer 0** is your canvas — swap it with anything:

```dart
// Gradient (current default)
Container(
  decoration: const BoxDecoration(
    gradient: LinearGradient(
      colors: [Color(0xFF4F46E5), Color(0xFF7C3AED)],
      begin: Alignment.topLeft,
      end: Alignment.bottomRight,
    ),
  ),
),

// Solid colour
Container(color: Colors.black),

// Asset image
Image.asset('assets/bg.jpg', fit: BoxFit.cover),

// Network image
Image.network('https://example.com/bg.jpg', fit: BoxFit.cover),
```

**Layer 1** is `AuthLoginScreen` with a transparent background so it shows through:

```dart
AuthLoginScreen(
  config: const AuthUIConfig(
    backgroundColor: Colors.transparent,
    headerBackground: BoxDecoration(color: Colors.transparent),
  ),
  onSuccess: () { /* navigate to home */ },
)
```

## Customisation options used in this example

### Auth methods

```dart
AuthUIConfig(
  enableEmailPassword: true,  // show email/password tab
  enableOtp: true,            // show phone OTP tab
  enableGoogle: true,         // show Google tab
)
```

### Background (transparent to show Stack layer behind)

```dart
AuthUIConfig(
  backgroundColor: Colors.transparent,
  headerBackground: BoxDecoration(color: Colors.transparent),
)
```

### Header logo (optional — commented out in example)

```dart
// Replace lock icon with your own:
headerLogoWidget: Image.asset('assets/logo.png', width: 52, height: 52),

// Or hide it completely:
headerLogoWidget: SizedBox.shrink(),
```

### Header text styles (optional — commented out in example)

```dart
headerTitleStyle: TextStyle(
  color: Colors.white,
  fontSize: 28,
  fontWeight: FontWeight.w800,
),
headerSubtitleStyle: TextStyle(color: Colors.white70),
```

### Decorative circles (optional)

```dart
showHeaderDecorations: false, // hides the subtle circle overlays
```

### Color scheme (optional — commented out in example)

```dart
colorScheme: AuthColorScheme(
  primary: Color(0xFF0D9488),           // accent colour
  buttonColor: Color(0xFF0D9488),       // button fill
  inputFocusBorderColor: Color(0xFF0D9488), // focused border
  linkColor: Color(0xFF0D9488),         // "Forgot password?" etc.
  cardColor: Colors.white,              // form card background
  textPrimaryColor: Colors.black87,     // input text colour
),
```

## Navigation flow

This example uses **imperative navigation** (no named routes):

```
_Root (LoginScreen)
  │
  ├─ onSuccess ──────────────────→ HomeScreen
  │                                  │
  │                                  └─ logout ──→ _Root (LoginScreen)
  │
  └─ onSignupTap ────────────────→ SignupScreen
                                     │
                                     ├─ onSuccess ──→ HomeScreen
                                     └─ onLoginTap → back to LoginScreen
```

## What gets stored in Firebase

| Action | Firebase Auth | Firestore (`users` collection) |
|---|---|---|
| Email signup | ✅ email + UID | ✅ `{ uid, name, email }` |
| Email login | ✅ | ❌ |
| Google sign-in | ✅ email + UID | ✅ `{ uid, email, name }` |
| OTP login | ✅ phone + UID | ❌ |

## Troubleshooting

**`firebase_options.dart` not found**  
Run `flutterfire configure` inside the `example/` folder.

**Google Sign-In not working on Android**  
Make sure the SHA-1 fingerprint of your debug keystore is added in the Firebase console under **Project Settings → Your apps → Android app**.

**Phone OTP not working**  
Ensure the **Phone** sign-in method is enabled in Firebase Authentication and your device has an active SIM / internet connection.

**Firestore permission denied**  
Set Firestore rules to test mode temporarily:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```
